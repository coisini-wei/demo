#### 静态语言
程序运行是的结构应是确定的。如java、c、c++


#### 反射流程
正常方式：类的全路径名称-> 通过new进行实例化 -> 获取实例化后的对象
反射方式: 实例化后的对象 -> 调用getClass方法 -> 得到全类名

#### 反射机制的应用场景
- 运行时判断一个对象所属的类
- 运行时判断一个对象所拥有的方法、属性
- 在运行时构建任意一个对象
- 运行时获取泛型信息
- 运行时生成代理
...

##### 优点
可以动态实时编译，体现出强大的灵活性
##### 缺点
对性能有影响。反射的过程和new相比，效率没有那么高
